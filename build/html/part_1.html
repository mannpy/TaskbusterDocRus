Часть I – Рабочая среда и начало проекта Django¶

Создание хорошей рабочей среды очень важно для вашей производительности. Может быть, это займет некоторое время, для того чтобы настроить ее, но когда это хорошо сделано - то стоит потраченного времени!

Например, виртуальная среда поможет вам держать ваши пакеты организованными и контролируемыми, а хороший текстовый редактор поможет вам с именами переменных, методами пакетов, подсветкой синтаксиса.

Да, и если вы не читали введение, этот Django учебник в актуальном состоянии.

Мы будем использовать Django 1.8 и Python 3. Это не означает, что вы не можете следовать учебнику, если вы используете старую версию. Это означает только то, что вам придется самим находить и устранять, возникающие ошибки, чтобы решить проблемы, которые я не могу здесь предвидеть.

Создание вашей рабочей среды¶

Для того, чтобы следовать этому Django учебнику вам понадобится Python3, менеджер пакетов pip3 и следующие пакеты: virtualenv и virtualenvwrapper.

Комплект инструментов TaskBuster: Python3, pip3, virtualenv, virtualenvwrapper, Git и Sublime Text

Если вы используете Mac OS X вы можете посмотреть этот пост: Установить Python 3 на Mac OS X и использовать virtualenv и virtualenvwrapper

Вам также понадобится хороший текстовый редактор для управления всеми файлами Django, которые мы будем создавать. Я рекомендую Sublime Text, но если вы чувствуете себя комфортно с другим текстовым редактором - то вперед! Смотрите здесь: как установить и настроить Sublime Text 3 для Django.

Несмотря на то, что мы будем говорить об этом в следующем посте, нам будет нужен Git, система управления пакетами. Если более чем один пользователь будет управлять кодом, то я рекомендую вам использовать Git репозиторий, такие как GitHub или Bitbucket (последний из которых предлагает частные репозитории для бесплатных аккаунтов). Вы можете узнать основы о системе Git по этой ссылке Git Tutorial.

Хорошо, теперь мы готовы начать наш супер-крутой проект Django

Установка Django 1.8¶

Во-первых, мы создадим виртуальную среду с Python3, по умолчанию как Python. В вашем терминале, введите:

$ which python3

чтобы узнать ваш путь к python3. В моем случае это было /usr/local/bin/python3. Далее, создайте виртуальную среду для разработки, и укажите Python3 как Python по умолчанию (напомню, что с virtualenvwrapper виртуальные среды создаются в папке ~/.virtualenvs, независимо от рабочей папки):

$ mkvirtualenv --python=/usr/local/bin/python3 tb_dev

Измени свой путь к Python, если он отличается от моего, а также измените имя виртуальной среды tb_dev на другой, если хотите. Вы увидите, что эта виртуальная среда теперь активна - командная оболочка будет выглядить как то так: (tb_dev)[user@host]$.

Вы можете отключить виртуальную среду написав:

$ deactivate

и активировать заново так:

$ workon tb_dev

Обратите внимание, что если вы работаете в этой среде, когда вы вводите python вы активируете python3, и при вызове pip, вы вызываете версию python3, pip3. Круто!

Готовы установить последнюю версию Django?! С активным tb_dev, введите:

$ pip install Django==1.8.13

Примечание: Вы можете проверить, есть ли новая версия, доступная здесь.
